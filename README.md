# 救急外来 振り返りシート PWA

スマホのホーム画面に追加して、ネイティブアプリのように使える振り返り記録アプリです。

## 同梱ファイル

| ファイル | 役割 |
|---|---|
| `index.html` | 本体（元のHTMLにPWA関連タグ・SW登録を追加） |
| `manifest.json` | アプリ情報（名前・アイコン・テーマ色） |
| `service-worker.js` | オフライン対応（キャッシュ制御） |
| `icon-192.png` / `icon-512.png` | アプリアイコン |
| `apple-touch-icon.png` | iOS ホーム画面用アイコン（180×180） |

## GitHub Pages への公開手順

### ① リポジトリを作る
1. GitHub にログインして **New repository** から新規リポジトリを作成
   - 名前は何でもOK（例：`furikaeri-sheet`）
   - **Public** に設定（Private だと Pages 無料枠が使えない場合あり）

### ② ファイルをアップロード
同梱の5ファイルをすべてリポジトリのルートに配置します。
- ブラウザから **Add file → Upload files** で全部まとめてドラッグ＆ドロップでOK
- `git` が使えるなら `git add . && git commit -m "init" && git push`

### ③ Pages を有効化
1. リポジトリの **Settings → Pages** を開く
2. **Source** を **Deploy from a branch** にして
3. **Branch** を `main` / root (`/`) に設定 → **Save**
4. 数十秒〜数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` でアクセス可能に

### ④ スマホのホーム画面に追加
- **iPhone (Safari)**：共有ボタン → 「ホーム画面に追加」
- **Android (Chrome)**：メニュー → 「ホーム画面に追加」またはインストールバナーから

ホームから起動すると URL バーなしの全画面で開きます。

## オフライン動作について

一度アクセスすると Service Worker がアプリ本体をキャッシュするので、以降は**電波がなくても起動・入力・保存が可能**です。
保存データは端末の `localStorage` に保存されるので、サーバー送信は発生しません。

## 更新（アプリを改修したとき）

1. 変更したファイルを GitHub に push
2. **`service-worker.js` の `CACHE_VERSION` を `'v1'` → `'v2'` に上げる**
   - これをやらないと、古いキャッシュが優先されて変更が反映されません
3. ユーザーが次に開いたとき（※一度ブラウザバック or リロードが必要なことも）新しい版が読み込まれる

## 注意事項

- **PIN コード** は `index.html` 冒頭の `const PIN_CODE = '0000'` に直書きされています。公開前に必ず変更してください。
  - ※クライアント側のJSに書いてあるため、本格的な機密保護にはなりません。CSV保護の「軽い蓋」程度と考えてください。
- **個人情報**：症例入力欄には氏名・生年月日などを書かない運用を。localStorage は端末ブラウザに残り続けます。
- **端末紛失時**：データは端末内に残るので、業務用端末以外での利用は慎重に。
- **Private リポジトリ**：GitHub Pages は Public なら無料で使えます。Private にしたい場合は有料プラン（Pro 以上）が必要です。
